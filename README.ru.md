# ZapisBOT — Telegram-бот записи (self-hosted)

White-label система онлайн-записи для сервисного бизнеса (барбершопы, груминг,
салоны красоты, стоматологии, массаж и т.д.) с **полноценной админкой прямо
в Telegram** — без веб-панели.

Запускаете у себя: вставляете токен бота, правите `config.yaml`, стартуете.

> Интерфейс бота — **на русском**: все сообщения, кнопки и команды.

---

## Возможности

- **Роли**: клиент, мастер, админ (определяются автоматически по Telegram id).
- **Клиент**: запись на услугу → выбор мастера → дата → время; просмотр, **перенос** (с лимитом) и отмена своих записей; **фото** услуги/мастера на шаге подтверждения.
- **Мастер**: видит только своё расписание на день; отмечает *пришёл* / *не пришёл*.
- **Админ-панель** (`/admin`, inline-кнопки):
  - 📊 **Статистика** — записи за день/неделю/месяц, доход, загрузка мастеров, отмены, неявки.
  - 📅 **Записи** — все (с пагинацией) / сегодня / по мастеру; отмена, **перенос**, выполнено, неявка.
  - 💈 **Услуги** — добавить / удалить (с подтверждением и проверкой) / вкл-выкл / цена / длительность / **фото** / привязка к мастерам.
  - 👨‍🔧 **Мастера** — добавить / удалить / рабочие часы / **разовые выходные** / **фото** / назначить услуги.
  - 🚫 **Чёрный список** — список, ручная блокировка/разблокировка.
  - ⚙️ **Настройки** — напоминания, буфер, лимиты отмен, правила неявок, retention. Значения **валидируются** (кривое число не сломает бота).
- **Автоматические слоты** — из рабочих часов, длительности услуги и буфера; пересечения исключены; минимальный запас по времени.
- **Защита от двойной записи** — частичный unique-индекс в БД (SQLite + PostgreSQL) не даёт две активные записи на одного мастера/время даже в гонке.
- **Уведомления** — создание / отмена / **перенос** записи, напоминания за 24 ч и 1 ч. Архитектура готова к SMS/WhatsApp.
- **Авто-блокировка** — 2 неявки → авто-ЧС; предупреждения при превышении лимита отмен.
- **Антифлуд** — throttling по пользователю.
- **Телефон** — необязательная кнопка «Поделиться контактом» сохраняет номер клиента.
- **Неизменяемый аудит-лог** — фиксируются все действия (создание/отмена/перенос/статусы/действия админов). На SQLite это гарантируется на уровне БД (триггеры блокируют UPDATE/DELETE), а не только в коде.
- **Retention + архив статистики** — выполненные/отменённые записи удаляются через 7 дней (настраивается), неявки хранятся дольше. Дневные итоги (количества + доход) архивируются **до** удаления, поэтому статистика не теряется.
- **Ежедневный бэкап БД** — авто-снимок SQLite (`backups/`) с ротацией.
- **Утренняя сводка мастеру** — список записей на день в 08:00 по местному времени.
- **Экспорт CSV** — админ выгружает все записи документом в чат (открывается в Excel).
- **Рассылка** — админ отправляет сообщение всем пользователям из панели.
- **О нас / контакты** — `/about` показывает адрес, телефон, карту, соцсети (из конфига).
- **Анти-спам / анти-бот** — throttling, лимит активных записей на клиента, пауза между записями, опционально «телефон до записи».
- **Усиление** — весь пользовательский текст HTML-экранируется (нет инъекций/поломки разметки), лимиты длины ввода, глобальный обработчик ошибок.
- **SQLite** по умолчанию; переход на **PostgreSQL** — одна строка.
- **Long-polling** — одна команда, работает где угодно (без домена/HTTPS).

## Требования

- **Рекомендуется Python 3.10+** (работает на 3.9; на 3.8 автоматически берёт `backports.zoneinfo`).
- Токен бота от [@BotFather](https://t.me/BotFather).
- Ваш числовой Telegram id (узнать у [@userinfobot](https://t.me/userinfobot)).

## Установка

```bash
# 1. Зависимости
pip install -r requirements.txt

# 2. Создать свой конфиг
cp config.example.yaml config.yaml      # Windows: copy config.example.yaml config.yaml

# 3. Отредактировать config.yaml:
#    - bot_token   (или переменная окружения BOT_TOKEN)
#    - admins      (ваш Telegram id)
#    - business_name, timezone, services, masters

# 4. Запуск
python bot.py
```

При первом запуске создаётся база, и `services`, `masters`, `rules` из
`config.yaml` загружаются в неё. Дальше всё управляется через `/admin` —
конфиг наполняет значения по умолчанию только один раз (новые ключи `rules`
добавляются, не затирая ваши правки).

---

## Конфигурация (`config.yaml`)

| Ключ | Значение |
|------|----------|
| `bot_token` | Токен BotFather (переопределяется env `BOT_TOKEN`). |
| `business_name` | Показывается клиентам. |
| `timezone` | Имя IANA, напр. `Europe/Moscow`. |
| `currency`, `show_prices` | Отображение цен. |
| `admins` | Список Telegram id с правами админа. |
| `database_url` | SQLite (по умолчанию) или DSN PostgreSQL. |
| `services` | Начальные услуги (название/длительность/цена). |
| `masters` | Начальный персонал (имя, опц. tg_id, часы, услуги). |
| `rules` | Буфер, шаг слота, горизонт, правила отмен/неявок, напоминания, retention, анти-спам (`max_active_bookings`, `booking_cooldown_seconds`, `require_phone`, `max_client_reschedules`). Редактируется в `/admin → Настройки`. |
| `contacts` | Адрес / телефон / карта / соцсети салона для `/about`. |

Все поля с комментариями — в `config.example.yaml`.

## Тесты

```bash
pip install -r requirements-dev.txt
pytest
```

Покрывают генерацию слотов + буфер, защиту от двойной записи, авто-ЧС,
лимиты отмен/переносов, корректность архива retention, неизменяемость аудита,
бэкап.

## Роли — как назначаются

- **Админ**: любой Telegram id из списка `admins:`.
- **Мастер**: пользователь, чей id совпадает с `tg_id` мастера (задаётся в конфиге или в `/admin → Мастера`).
- **Клиент**: все остальные.

## Переход на PostgreSQL

```bash
pip install asyncpg
```

```yaml
# config.yaml
database_url: "postgresql+asyncpg://user:password@localhost:5432/zapisbot"
```

Таблицы создаются автоматически при старте. (Для миграций схемы со временем
добавьте Alembic — модели в `app/models.py`.)

## Структура проекта

```
bot.py                  точка входа
config.example.yaml     шаблон конфига
app/
  config.py             загрузка/валидация config.yaml
  models.py             ORM SQLAlchemy (SQLite/PostgreSQL)
  db.py                 движок, сессии, сидинг, хранилище настроек
  slots.py              генерация слотов + буфер/пересечения

  blacklist.py          ручной + авто ЧС, лимиты отмен
  audit.py              append-only аудит-лог
  notifications.py      Telegram-уведомитель (готов к SMS/WhatsApp)
  scheduler.py          напоминания, retention, сводка, бэкап (APScheduler)
  middlewares.py        сессия БД на апдейт + роли + антифлуд
  keyboards.py          inline-клавиатуры
  states.py             FSM-состояния
  utils.py              время/форматирование/экранирование
  handlers/
    common.py           /start /help /cancel /about
    client.py           сценарий записи + мои записи + перенос
    master.py           расписание мастера + статусы
    admin.py            полная админ-панель
```

## Расширение

- **Новый канал уведомлений**: реализуйте `Notifier` в `app/notifications.py` и добавьте его в `setup()`.
- **Новые отчёты в админке**: добавьте callback в `app/handlers/admin.py` и кнопку в `keyboards.admin_main_kb()`.
- **Разовые выходные мастера**: `Master.days_off` хранит JSON-список ISO-дат (`["2026-07-01"]`); генерация слотов их учитывает.

---

## Примечания / ограничения (MVP)

- FSM хранится в памяти — рестарт сбрасывает незавершённые сценарии (но не записи).
- Однопроцессный long-polling — идеален для одного салона; работает на дешёвом VPS или домашнем ПК.
- Напоминания проверяются раз в минуту; сводка в 08:00, очистка в 03:00, бэкап в 03:30 (местное время).
